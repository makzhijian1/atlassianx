# ATL-006 Specification

## Behaviour

- Bruno request templates inherit Jira authentication from the collection settings.
- Individual request files do not define their own Basic Auth blocks.
- Bruno environment files contain only shared connection and auth values.
- Request-specific sample values live in the request file that uses them.
- Issue keys, epic keys, issue types, and JQL strings are request-local pre-request variables.
- The basic JQL sample searches for summaries containing `found by JQL`.

## Contracts

- The committed Bruno template collection remains under:
  - `setup/requests/bruno-jira-template/`
- Collection-level auth is configured once in:
  - `setup/requests/bruno-jira-template/collection.bru`
- Collection-level auth uses:
  - username: `{{JIRA_EMAIL}}`
  - password: `{{JIRA_API_TOKEN}}`
- The committed template environment contains only:
  - `JIRA_SITE_URL`
  - `JIRA_EMAIL`
  - `JIRA_API_TOKEN`
  - `JIRA_CLOUD_ID`
  - `GRAPHQL_URL`
- The generated project Bruno environment file contains only the same shared values.
- REST request files inherit auth from the collection.
- GraphQL request files inherit auth from the collection.
- Request-local sample values are declared in request pre-request variables.
- Request-local variables include, where relevant:
  - `ISSUE_KEY`
  - `EPIC_KEY`
  - `ISSUE_TYPE`
  - `JQL`
- The search JQL sample value is:

```text
summary ~ "found by JQL" ORDER BY updated DESC
```

- `JIRA_PROJECT_KEY` is not a Bruno environment variable unless a later request proves it is truly shared connection state.
- Create issue requests should use request-local variables for sample issue type and any project key needed by the payload.
- Environment values generated by `make init-jira-account PROJECT=<project>` must not include request-specific values.
- No API token is printed to stdout.

## Inputs / Outputs

- Inputs:
  - Existing Bruno Jira template collection.
  - Project `.env` values from `ATL-005`.
- Outputs:
  - Bruno collection with inherited auth.
  - Lean Bruno environment files.
  - Request files with pre-request variables for request-specific sample inputs.
  - Updated docs describing the variable boundary.

## Invariants

- `projects/` remains ignored local user state.
- Real Jira credentials are never committed.
- Request templates contain safe sample values only.
- Shared environment variables are limited to cross-request connection/auth values.
- Request-local values remain easy to edit inside Bruno.
- Existing REST and GraphQL request coverage is preserved.

## Edge Cases

- A request needs `JIRA_PROJECT_KEY` for payload construction.
- A request has no request-local variables.
- A generated project environment already contains old request-specific values from `ATL-005`.
- Users have edited local project Bruno collections.
- Bruno behavior differs between inherited collection auth and request-level auth.

## Failure Semantics

- Existing local project request files are not overwritten by `make init-project`.
- Re-running `make init-jira-account PROJECT=<project>` rewrites the generated Bruno environment to the new shared-value shape.
- Missing shared env values fail through Bruno request execution or existing Jira account validation.
- Implementation must not print `JIRA_API_TOKEN` while generating or inspecting environment files.
